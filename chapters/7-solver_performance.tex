\chapter{Solver Performance}\label{chap:results_performance}

This chapter evaluates the performance of the proposed solver through a series of simulation studies and analysis. First, the solver is compared with results reported in previously published methods. The chapter then presents an extensive testing framework to evaluate the method's robustness and convergence behavior across a large number of randomly generated leak configurations. The chapter further examines scenarios where the number of leaks is unknown and evaluates how estimated leak locations can still reduce the search area in practical leak localization. Finally, the solver's sensitivity to modeling errors and measurement noise is investigated. 

\section{Comparison Against Peralta et al. (2023)}

A method for identifying two leaks based on steady-state measurements is presented in \cite{peralta2023}. The publication also includes simulation results for five test cases and provides sufficient information to replicate them using the method proposed in this work. \begin{equation*}
    \ell=160.76, \quad \theta = 583.32.
\end{equation*}

\begin{align*}

q_0^{(1)} &= 13.97\times 10^{-3}, &

\section{Simulation Results}

An extensive test framework is designed to test the performance of the proposed method. The goal is to provide empirical results on the solver's performance across a wide range of leak configurations and pipe characteristics. The structure of the framework is shown in \autoref{fig:test-framework}. For a given number of leaks, a test suite consists of running many test cases as presented in \autoref{alg:test_suite}.\\

\begin{figure}[h!]
    \centering
    \input{figures/tikz_test_framework}
    \caption[Structure of the simulation test framework]{Structure of the simulation test framework. A test suite tests a specific number of leaks and consists of $N_\text{cases}$ test cases. Each test case samples a leak configuration and a pipe characteristic, then simulates measurements. For each test case, the solver is executed $N_\text{trials}$ times, with different initial values.}
    \label{fig:test-framework}
\end{figure}

\begin{algorithm}[ht]
\caption{Test Suite for $n$ Leaks}
\label{alg:test_suite}

\DontPrintSemicolon
\SetKwFunction{samplelc}{SampleLeakConfiguration}
\SetKwFunction{sampleig}{SampleInitialGuess}
\SetKwFunction{genmes}{GenerateMeasurements}
\SetKwFunction{lmsolve}{Solve}
\SetKwFunction{append}{Append}
\SetKwFunction{keys}{Keys}
\SetKwFunction{store}{Store}
\SetKwProg{Fn}{Function}{:}{}
\SetKwInput{KwNote}{\normalfont Note}
\SetKwRepeat{Try}{try}{until}
\For{$c \gets 1$ \KwTo $N_\mathrm{cases}$}{
    $\mathbf{x}_c^* \gets$ \samplelc{$n,\ell,\delta,c_{\min},c_{\max}$} \tcp*[r]{\autoref{alg:SampleLeakConfiguration}}
    $\theta \gets \mathcal{U}(\theta_{\min},\ \theta_{\max})$ \;
    $S_c \gets \{\mathbf{x}_c^* : 0\}$ \tcp*[r]{map: solution $\mapsto$ count}
    $\mathcal{M} \gets$ \genmes{$\mathbf{x}_c^*, \theta,m,R_{\Delta h},h_0$} \tcp*[r]{\autoref{alg:generate_measurements}}
    \For{$t \gets 1$ \KwTo $N_{\mathrm{trials}}$}{
        $\mathbf{x}_0 \gets$ \samplelc{$n,\ell,\delta,c_{\min},c_{\max}$} \tcp*[r]{\autoref{alg:SampleLeakConfiguration}}
        $\mathbf{x} \gets$ \lmsolve{$\mathcal{M},\theta,\mathbf{x}_0$} \tcp*[r]{\autoref{sec:solver_design}}
        \eIf{$\exists\mathbf{x}' \in S_c$ \text{such that} $\|\mathbf{x}-\mathbf{x}'\|_\infty < \varepsilon$}{
            increment count of $S_c(\mathbf{x}')$\;
        }{
            add $\mathbf{x}$ to $S_c$ with count $1$\;
        }
    }
    \store{$\mathbf{x}_c^\ast, S_c$}
}
\end{algorithm}

By running $N_\mathrm{cases}$ different, randomly generated test cases, the test suite covers a wide range of configurations and provides a good picture of the method's performance on simulated measurements. For simplicity, the pipe is normalized to unit length ($\ell=1$) and kept constant across test cases.\\

For each test case, the solver is executed $N_{\mathrm{trials}}$ times from independently sampled initial guesses. This allows the solver's sensitivity to the initial value to be assessed. If the solver consistently converges to the same solution across trials, the problem instance can be considered robust to initialization. If different solutions are obtained depending on the initial guess, the test case may exhibit multiple local optima or an ill-conditioned solution space.\\

\newpage
During each test case, all distinct solutions returned by the solver are stored in a map $S_c$, which records how often each solution is obtained.  Two solutions are considered similar if they differ by less than a small tolerance $\varepsilon$ in the infinity norm,
\begin{equation*}
\mathbf{x}_1 \sim \mathbf{x}_2
\ \Leftrightarrow \ 
\|\mathbf{x}_1 - \mathbf{x}_2\|_\infty < \varepsilon.
\end{equation*}
The map is initialized with the true solution, so that similar solutions are exactly mapped to it.\\

The experiment is limited to testing pipe systems with $ n=2, 3, 4, 5$ leaks, where the test suite for each number of leaks consists of $N_\text{cases}=1000$ test cases. The remaining parameters are constant across all test suites and summarized in \autoref{tab:sim_params}.

For the test suite with two leaks, all test cases converged to exactly one solution, which all corresponded to the true leak configuration. For five leaks, the solver no longer performs reliably. In all of these cases, only a negligible portion included the true leak configuration, while a larger portion included a close solution. In total, the solver identified a close solution in 27.8\% of the test cases while the true solution was only found in 3.7\% of the cases.\\


\subsection{False Detection Rate}
For a given test case, $\mathbf{x}^*_c$ is the true solution, and $S_c$ is the stored solution counts. Let the number of true positives (TP) and false positives (FP) for a given test case be defined as the number of trials that converged to the true solution or a false solution. Specifically,
\begin{align*}
    TP_c &:= S_c(\mathbf{x}_c^*), \\
    FP_c &:= \sum_{\tilde{\mathbf{x}}\in \{S_c \setminus \mathbf{x}_c^*\} } S_c(\tilde{\mathbf{x}}).
\end{align*}
The false detection rate (FDR) can then be defined as
\begin{equation*}
    FDR:=\dfrac{\sum_{c} FP_c}{\sum_{c} (TP_c + FP_c)},
\end{equation*}
which is a metric indicating how often the solver produces an incorrect result.\\

\autoref{tab:fdr} shows the FDR from the test suites for different numbers of leaks. It can be seen that for pipe systems with two or three leaks, the solver rarely produces a wrong solution. For four leaks, most solutions are correct, and the solver's results remain useful for identifying leaks. For five leaks, almost all solutions are incorrect with respect to the tight solution bound of $\varepsilon$ used to construct $S_c$.

\subsection{Error Magnitude}
\label{sec:err_mag}

The results in the previous sections quantify how often the solver converges to the correct solution. However, when the solver converges to an incorrect solution, it is also relevant to understand how far this solution deviates from the true leak configuration. This section investigates the magnitude of the error for all non-true solutions, $\tilde{\mathbf{x}} \nsim \mathbf{x}^*$, in order to assess how inaccurate an incorrect estimate may be in practice.\\

Two error metrics are introduced to quantify the deviation between an incorrect solution ($\tilde{\mathbf{x}}$) and the true solution ($\mathbf{x}^*$). The first metric is the mean absolute error across all leak locations,
\begin{equation*}
e^{z}_{\text{mean}} := \frac{1}{n}\sum_{i=1}^{n} |\tilde z_i - z_i^*|,
\end{equation*}
which measures the average positional deviation of the estimated leaks. The second metric is the maximum absolute error across leak locations,
\begin{equation*}
e^{z}_{\max} := \max_{i} |\tilde z_i - z_i^*|,
\end{equation*}
which measures the largest deviation among the estimated leak positions. Together, these metrics provide information about both the overall accuracy of an incorrect solution and the worst-case deviation in leak location.\\

\autoref{fig:z_error} shows the distribution of $e^{z}_{\text{mean}}$ and $e^{z}_{\max}$ for all incorrect solutions obtained across all test cases. The figure only includes results for $n=4$ and $n=5$, as these were the only test suites that produced a notable number of incorrect solutions.\\

\begin{figure}[ht]
    \centering
    \includesvg[width=\linewidth]{plots/plot_z_error}
    \caption[Distribution of the mean and maximum leak error]{Distribution of the mean leak error $e^{z}_{\text{mean}}$ and maximum leak error $e^{z}_{\max}$ for all incorrect solutions $\tilde{\mathbf{x}} \nsim \mathbf{x}^*$ obtained across test cases with $n=4$ and $n=5$ leaks.}    
    \label{fig:z_error}
\end{figure}

From the figure, it can be seen that the average mean error is $0.007$ for four leaks and $0.014$ for five leaks. Recall that the pipe length is normalized to one, which implies that the typical deviation in leak position is relatively small, indicating that many incorrect solutions are relatively close to the true configuration.\\

Some test cases produce substantially larger errors for individual leaks. Most maximum observed leak errors for four leaks are below 0.1, while for five leaks, a significant number exceed 0.1. Such deviations correspond to a significant fraction of the pipe length, rendering such solutions likely useless. These results indicate that while many incorrect solutions remain close to the true configuration, a subset deviates considerably and may offer limited practical value.\\

\section{Results when the Number of Leaks is Unknown}
\label{sec:n_not_known}

In practice, it is unlikely that the number of leaks in a pipe can be known. Let $\hat n$ be the assumed number of leaks in a pipe. Running the solver under this assumption will produce leak locations $\hat{z} _ k$, $ k = 1,\dots,\hat{n} $. Given the true leak locations $z^*_i$, the smallest error between an estimated and true leak location is given by
\begin{equation*}
    e^z_{\min} := \min_{i,k}|z^*_i-\hat z_k|.
\end{equation*}
Assuming some known upper bound of this error, a small $e^z_{\min}$ can narrow the search distance for a true leak. The search distance $d_s$ is given by
\begin{equation}
\label{eq:search_dist}
    d_s := 2 \hat{n} e^z_{\min}
\end{equation}
and illustrated by \autoref{fig:search_dist}.\\

\begin{figure}[ht]
    \centering
    \input{figures/tikz_search_distance}

    \label{fig:search_dist}
\end{figure}

\FloatBarrier

For each $n\in\{3,4,5,6\}$, 1000 different random leak configurations were simulated. The solver was then applied to each configuration with an assumed number of leaks, $\hat {n} \in \{2, 3\}$. This produced a distribution of $e^z_{\min}$ values across the different leak configurations, as shown in \autoref{fig:plot_est_n}.\\

\begin{figure}[ht]
    \centering
    \includesvg[width=\linewidth]{plots/plot_est_n}

From the plot, it can be seen that an estimated leak location $\hat{z}$ is often close to a true leak location. For $\hat{n} = 3$, the error is generally smaller than for $\hat{n} = 2$, which is expected since assuming a higher number of leaks yields more freedom for the solver. The downside of assuming more leaks is that the search distance grows larger. It should be noted that more advanced methods that combine results obtained by assuming different values of $\hat n$ might further minimize $d_s$. A design of such methods is beyond the scope of this work and will not be investigated.

\section{Sensitivity Analysis of Resistance Coefficient}
\label{sec:mod_err}

Let the estimate of \gls{theta} be given by 
\begin{equation*}
    \hat{\theta} = \theta + \delta\theta
\end{equation*}
where $\delta\theta$ is the modeling error. The relative modeling error is given by $\frac{\delta\theta}{\theta}$. To evaluate the solver's sensitivity with respect to the resistance coefficient, a measurement matrix \gls{M} is generated using the true value of $\theta$, and the solver is run with $\hat\theta$ for a range of errors. For the simple case of two and three evenly spaced leaks and small perturbations of $\delta\theta$, the resulting leak locations from the solver are given in \autoref{fig:theta_sens_comparison}.

\begin{figure}[h]
    \centering
    \includesvg[width=\linewidth]{plots/plot_theta_sens_2_and_3_leaks}
    \caption[Estimated leak locations for relative errors in $\theta$]{Estimated leak locations for different relative errors in $\theta$ for simple configurations of two and three leaks.}
    \label{fig:theta_sens_comparison}
\end{figure}

\FloatBarrier

The plot shows that the estimated leak locations are highly sensitive to perturbations in \gls{theta}. For these solutions, the solver converges at relatively high residuals, as shown in \autoref{fig:residuals_theta_sens}. In this case, the problem with a highly sensitive parameter can be addressed by running the solver over a wide range of values. The true solution and true value of \gls{theta} will then yield a smaller and detectable residual. \\

\begin{figure}[ht]
    \centering
    \includesvg[width=\linewidth]{plots/plot_residuals_theta_sens}
    \caption[Residual value at solutions for relative errors in $\theta$]{Value of residual at converged solutions for different relative errors in $\theta$.}
    \label{fig:residuals_theta_sens}
\end{figure}

Local sensitivity can also be analyzed through the derivative, $\frac{\partial \mathbf{x}}{\partial \theta}$. The residual function from \cref{eq:residual} can be expanded to $r(\mathbf{x},\theta)$. A small modeling error $\delta\theta$ will cause an error in the solution $\delta\mathbf{x}$. Using a first-order Taylor expansion yields
\begin{equation*}
r(\mathbf{x}^\ast+\delta \mathbf{x},\theta+\delta\theta)
\approx
r(\mathbf{x}^\ast,\theta)
+
J(\mathbf{x}^\ast) \delta \mathbf{x}
+
\dfrac{\partial r(\mathbf{x}^\ast,\theta)}{\partial \theta} \delta\theta.
\end{equation*}

Given that $r(\mathbf{x}^\ast,\theta)=0$, the parameter change that best compensates a small error in $\theta$ is therefore found from
\begin{equation*}
    J(\mathbf{x}^\ast) \delta \mathbf{x}+\dfrac{\partial r(\mathbf{x^\ast},\theta)}{\partial \theta} \delta\theta \approx 0.
\end{equation*}
The local sensitivity is then solved in the least-squares sense:
\begin{align*}
    J(\mathbf{x}^\ast)\dfrac{\partial \mathbf{x}}{\partial \theta}
    &=
    - \dfrac{\partial r(\mathbf{x}^\ast,\theta)}{\partial \theta}\\
    \dfrac{\partial \mathbf{x}}{\partial \theta}
    &=
    -\left(J(\mathbf{x}^\ast)^T J(\mathbf{x}^\ast)\right)^{-1}
    J(\mathbf{x}^\ast)^T
    \dfrac{\partial r(\mathbf{x}^\ast,\theta)}{\partial \theta}.
\end{align*}

This quantity measures how much the estimated leak parameters must change to compensate for a small modeling error in $\theta$. A large value, therefore, indicates that the inverse problem is ill-conditioned with respect to $\theta$.\\

The sensitivity can be computed from the analytical expression for $J(\mathbf{x}^\ast)$ derived in \autoref{sec:der_jacobian} and approximating the derivative with respect to \gls{theta} numerically with
\begin{equation*}
    \dfrac{\partial r(\mathbf{x}^\ast,\theta)}{\partial \theta}
    \approx
    \dfrac{
        r(\mathbf{x}^\ast,\theta+\varepsilon_\theta)
        -
        r(\mathbf{x}^\ast,\theta)
    }{\varepsilon_\theta},
\end{equation*}
where $\varepsilon_\theta$ is a small perturbation.\\

In the case of two leaks, the sensitivities are estimated to be
\begin{equation*}
    \dfrac{\partial \begin{bmatrix}
        L_1 & L_2
    \end{bmatrix}^\top}{\partial \theta} = \begin{bmatrix}
        365 \\ 783
    \end{bmatrix},
    \qquad
    \dfrac{\partial \begin{bmatrix}
        c_1 & c_2
    \end{bmatrix}^\top}{\partial \theta} = \begin{bmatrix}
        297 \\ -297
    \end{bmatrix},
\end{equation*}

which indicates extreme sensitivity. A perturbation in $\theta$ of 0.01 yields a change in $L_1$ of 3.65, which is many times larger than the length of the pipe.

\section{Sensitivity Analysis of Measurements}
\label{sec:meas_err}

Let $\hat{\mu}$ denote an element of \gls{M}, representing a single measurement of either an inlet or outlet flow or head measured at a steady-state operating point. Here, $\mu$ denotes the true physical quantity. The measurement noise is modeled as a relative normal distribution according to
\begin{equation*}
\hat{\mu} = \mu(1+\epsilon), \quad \epsilon \sim \mathcal{N}(0,\sigma_\mu^2),
\end{equation*}
where $\sigma_\mu$ denotes the relative standard deviation.\\

A Monte Carlo simulation is performed to evaluate how the solver behaves when noise is introduced into the measurements. For a given $\sigma_\mu$, the solver is executed 100 times using independently sampled noisy measurements on a simple leak configuration containing two leaks. The resulting mean and standard deviation of the estimated leak locations for different values of $\sigma_\mu$ are shown in \autoref{fig:meas_noise}.\\

\begin{figure}[ht]
    \centering
    \includesvg[width=\linewidth]{plots/plot_meas_sens.svg}
    \caption[Estimated leak locations for different measurement noise levels]{Estimated leak locations for different measurement noise levels $\sigma_\mu$ in a two-leak configuration. The results show the mean and standard deviation across 100 Monte Carlo simulations. Dashed lines indicate the true leak locations.}
    \label{fig:meas_noise}
\end{figure}

The results show that the mean estimated leak locations remain close to the true leak locations across the range of relative measurement noise. The standard deviation of the estimates increases rapidly with $\sigma_\mu$, indicating a large spread across individual simulations. Even with a relative standard deviation of only 0.5\%, the estimated leak location can deviate by more than 30\% of the pipe length in some cases. These results indicate that the solver is highly sensitive to measurement noise.
